OVERVIEW

The Community Dashboard Editor (CDE), which is integrated in the Pentaho User Console (PUC), is a graphical tool for creating, editing and previewing Pentaho dashboards.

Creating dashboards

The Files and settings menu allows you to create, save and rename dashboards.

Designing a dashboard

For designing your dashboard CDE offers three perspectives:

  • Layout: For designing the layout of your dashboard, from scratch or from a template. While defining the layout you can apply styles and add HTML elements as text or images.
  • Components: For adding and configuring the different components that make up your dashboard: Visual components (textboxes, tables, charts, etc.), Parameters and Scripts.
  • Data Sources: For defining the data sources used by the components

Previewing a dashboard

The Preview option is the shortcut for testing the L&F and behavior of your dashboard as you are working.

LAYOUT

The layout of dashboards relies on several standard well known technologies:

Besides, CDF incorporates Blueprint, a CSS framework which aims to cut down on your development time. Blueprints offers:

  • A rich set of classes which allows flexible layouts
  • A predefined 24-column layout which allows an easy way for building the dashboard layout
  • Cross browser compatibility
  • Good mobile rendering

Designing the Layout:

A dashboard layout can be created by a simple combination of rows and columns along with some HTML blocks and possibly some images.

You do all this in the Layout perspective. On the top left area you have a main toolbar that allows you to add, move or delete the different building blocks: rows, columns, spaces, HTML blocks and images.

As you build the layout you will see the structure of nested rows and columns below the toolbar.

If you select one of the elements on that structure, the properties area on the right is refreshed. There you can configure the element as you wish.

For designing your dashboard CDE offers three perspectives:

Dimensions:

You can assign the width of a column with the following properties:

  • Span size, where you will draw the components.
  • Prepend size and Append size for making some space between columns.

While these properties are specified in Blueprint units, the height for the rows has to be supplied in pixels. Because of Blueprint, in every row the sum of Prepend size, Append size and Span size must be 24!

Look and Feel:

In the Layout perspective you may also provide some L&F properties as for example a background color, and style for the corners of rows and columns.

Besides, if you add a CSS resource, you can apply any of the styles defined there to any of the elements in your layout by typing it in the Css Class property.

  • HTML for describing web pages
  • CSS for controlling the style and layout
  • JavaScript for adding interactivity
  • jQuery for simplifying all those tasks

DATASOURCES

Dashboards can be populated with data coming from a variety of sources, namely:

  • Databases.
  • Mondrian Cubes.
  • Pentaho Metadata.
  • XML files.
  • Scripting: You can define an ad-hoc data source (for example a small table) just by typing a piece of code.
  • Kettle transformations: Kettle allows you to get data from other sources as for example Excel sheets or web services.

The data sources can be parametrized, which makes your data totally versatile.

Adding datasources to a dashboard:

In the Datasources perspective you to define the origin of the data that will be used for populating your dashboard. Basically, you have to specify where the data comes from and how it looks like.

This is how you do it:

  • From the left menu expand the category where your data belongs. For example if you have to query a Mondrian cube, expand the category MDX Queries.
  • From the list of data source types, click the one that fits your needs, for example mdx over mondrianJndi. A new data source will be added in the pane located in the middle of the screen.

When you select a data source the right pane displays its properties. It's in that pane where you define the data source.

Depending on the kind of data source selected, the list of properties may vary. In general, most of the data sources share the following properties:

  • Name of the data source. The name must be representative because you will need it when defining a component that uses it.
  • JNDI name or JDBC parameters (host, user, etc.)
  • Query: This property is where you type the query (SQL, MDX, etc. depending on the kind of data source)
  • Column configuration: This property allows you to rename columns returned by the query, or create new calculated columns.
  • Output options: This property allows you to reorder or keep a subset of the columns returned by the query.

COMPONENTS

The Components are the central elements of a dashboard. There are three kinds of them:

  • Visual components: The components that are displayed in your dashboard. Examples of visual components are textboxes, tables, charts (pies, bars, timelines, etc.), selectors (radio buttons, date pickers), OLAP views, reports, etc.
  • Parameters: Represent values that are shared by the components. The parameters are essential for the interaction.
  • Scripts: Pieces of JavaScript code that let you customize the look and feel or behavior of other components.
  • XML files.

Adding and configuring components:

Whether you need to add a visual component, a parameter or a script, you need to access to the Component perspective.

  • From the left menu expand the category where your component belongs.
  • For example if you want to add a Date input component, expand the category Selects.
  • From the list click the one that interest you. A new component will be added in the pane located in the middle of the screen.
  • When you select a component the right pane displays its properties. It's in that pane where you define the component.

The default Properties pane will display the main properties. Click on the Advanced Properties button for the full list (which depends on the kind of the selected component).

In general most components share the following properties:

  • executeAtStart. Flag to indicate if the component is to be rendered or not when the dashboard is loaded.
  • HtmlObject. Where in the layout the component is going to be displayed.
  • Datasource property: Which data source will be used (only for the components that display data, e.g. a pie chart). If the datasource expects parameters, you may/should configure the Parameters property as well.
  • preExecution. Function that prepares the data before the update of the component.
  • postExecution. Function that is executed after the update of the component.
  • listeners. List of variables that fire the update of this component.

In the case of selectors you also have these options:

  • preChange (opt.). Function to be applied after the user clicks or select an option. It can be used to validate the input.
  • postChange (opt.). Function to be applied after the parameter has changed its value.
  • Parameter: the name of the parameter that the selector will affect.

QUICKSTART

Can't wait for creating your first dashboard? Follow these simple steps:

Imagine

Think of how your dashboard will look like and how it will behave. Draw a sketch on paper for future reference.

Log into the PUC, and launch the CDE

You can launch the editor from the PUC home page, from the PUC menu (File --> New --> CDE Dashboard), or by clicking on the CDE icon in the PUC toolbar.

Save the dashboard

Give the dashboard a name and save it in the solution folder of your choice. Close the dashboard.

Edit the dashboard

Refresh the console by clicking F5, look for the dashboard, right-click it and select Edit. The dashboard editor will appear.

Design the Layout

Select the Layout perspective and define the dashboard layout according to your sketch.

Add the datasources

Select the Datasource perspective and define the datasources that will feed your dashboard.

Add and configure the components

Select the Component perspective and add the components that make up your dashboard: the visual elements, the parameters, and eventually some scripts.

Preview your work

Click on Preview regularly to see hoy your work is progressing. Repeat steps 5-8 until you are satisfied with your final dashboard! Don't forget to save your work from time to time.
